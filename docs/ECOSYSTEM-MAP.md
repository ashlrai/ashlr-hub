## The composition flywheel — who provides / consumes what

ashlr-hub is the **conductor**. It enrolls repos' backlogs, dispatches sandboxed agent swarms across multiple backends, and deposits proposal-only diffs into a human-gated Approval Inbox. Everything else either feeds the fleet a capability or is a product the fleet can build. The flywheel: the fleet uses its own ecosystem to make itself cheaper, safer, and more capable, then ships more of the ecosystem.

```
                          ashlr-hub  (conductor: daemon/loop/goal/fleet/inbox)
                              |  consumes capabilities, deposits proposals
        ┌──────────────┬──────┴────────┬─────────────────┬───────────────────┐
   Tier 1 (fix self)  Tier 2 (observe)  Tier 3 (build)   tool layer          substrate
   phantom-secrets    ashlr-pulse       webfetch         ashlr-plugin        @ashlr/core-efficiency
   ashlrcode/workbench stack             morphkit         (40 MCP tools)      (snipCompact/genome/budget)
   core-efficiency    prompt-trackr     ashlr-md
   binshield
```

### Provider → consumer edges (the load-bearing compositions)

| Provider | Provides | Consumed by | Edge |
|---|---|---|---|
| **@ashlr/core-efficiency** | snipCompact / autoCompact / genome RAG / provider budgets / token counting | ashlr-plugin, ashlrcode | The compression substrate under the whole tool layer |
| **ashlr-plugin** | 40 `ashlr__*` MCP tools + genome scribe | every host agent (Claude Code, Codex, Cursor), ashlr-workbench, ashlr-hub sandboxes | The token-efficient tool surface the fleet routes work through |
| **ashlrcode / ashlr-workbench** | full executor agents (45 tools / 4 wired OSS agents) | ashlr-hub fleet (executor backend) | Real local muscle instead of raw `runGoal` |
| **phantom-secrets** | real API-key injection at the network edge; 25 `phantom_*` MCP tools | ashlr-hub daemon+sandboxes, ashlr-stack (hard dep) | Real secrets never enter config/logs/LLM context |
| **binshield** | dependency-safety scan API + GitHub Action + SARIF | ashlr-hub trust gate | Kills blind dep-bumps; gates autonomous dep work |
| **ashlr-pulse** | OTLP GenAI-span ingest + Fleet Oversight control plane | ashlr-hub (emits telemetry), ashlr-plugin, all coding agents | The fleet's live mission-control dashboard + audit trail |
| **ashlr-stack** | one-command provisioning of 43 providers; 19 `stack_*` MCP tools | ashlr-hub (provisions infra for products it builds) | Wires a downstream project's whole backend |
| **prompt-trackr** | prompt-quality scoring (AES, 6 dims) + SDK/API | ashlr-hub self-improvement loop | Score + iterate the fleet's own prompts |
| **webfetch / morphkit / ashlr-md** | license-first image search / web→iOS / Markdown review surface | ashlr-hub product builds | Tier-3 building blocks for products the fleet creates |

### Composition bets (prioritized)

**Tier 1 — the fleet uses its own ecosystem to fix its own weaknesses:**
1. **phantom → fleet auth.** Inject real engine/API keys into the daemon + sandboxes; phantom tokens stay safe in config/logs/git. Fixes auth-in-daemon directly.
2. **ashlrcode / workbench → executor backend.** Dispatch fleet work to `ac` (45 tools, worktree isolation, real test loop) or the workbench's local-agent pool. Fixes local-execution quality.
3. **core-efficiency / plugin → the fleet's own token cost.** Run strategist/judge/invent prompts through snipCompact + genome + cacheBreakpoints. Slashes burn.
4. **binshield → dep-safety gate.** Scan every proposed dependency change before merge; risk feeds the trust gate.

**Tier 2 — observability + provisioning:** 5. **pulse → fleet telemetry.** 6. **stack → provisioning.** 7. **prompt-trackr → self-improvement.**

**Tier 3 — product-building blocks:** 8. **webfetch + morphkit + ashlr-md** as composable capabilities for products the fleet creates.

### ashlr-hub — *the conductor*

- **Stack:** TypeScript (ESM) on Node >=22. Build `tsc` → `dist/` + copy-assets; optional SEA binary. Vitest, ESLint 9. Zero runtime deps except `@modelcontextprotocol/sdk`. Sub-surfaces: Raycast extension, Tauri desktop app, terminal TUI.

- **Key modules:** `src/cli/index.ts` (68KB ~70-case router); `src/api/{index,core,plugin,types}.ts` (public npm surface — `applyProposal` deliberately NOT exported); `src/core/types.ts` (155KB); `src/core/config.ts` (`~/.ashlr/config.json`); `src/core/daemon/loop.ts` (66KB operator loop); `src/core/run/orchestrator.ts` (96KB) + run/router + sandboxed-engine + sandbox/{worktree,confine,policy}; `src/core/swarm/{runner,gate,sign,rollback}`; `src/core/inbox/{store,merge,apply}` (merge 69KB); `src/core/fleet/` (28 files: manager/judge, router, quota, self-improve, regression-sentinel, red-team, taste-critic); `src/core/foundry/provenance.ts`; `src/core/genome/`; `src/core/goals/+vision/+strategy/`; `src/core/ecosystem/map.ts`; `src/core/integrations/`; `src/core/web/` (Mission Control 127.0.0.1:7777); `src/core/plugins/`; `bin/ashlr`; `schema/config.schema.json`.
- **Public API:** CLI `ashlr` (~70 commands: setup, daemon, loop, goal(s), inbox, swarm, run, fleet, manager, vision, invent, comms, serve, eval, verify-safety, doctor, mcp, sandbox gc, tui, …); npm `@ashlr/hub` read-heavy API (loadConfig, loadIndex, buildOrientation, recall, listRuns/Swarms, estimateRun, listProposals/createProposal/pendingCount — **apply NOT exported**, callNativeTool, buildSnapshot); subpaths `/core`, `/types`, `/plugin`; native MCP tool registry + MCP gateway; local HTTP Mission Control at 127.0.0.1:7777; on-disk `~/.ashlr/` contract.
- **Entry / build / test:** `bin/ashlr` → `dist/cli/index.js`; dev `npm run dev` (tsx); build `npm run build`; SEA `npm run build:binary`; test `npm run test` / `npm run test:serial`; typecheck `npm run typecheck`; lint `npm run lint`; install from source `./install.sh`.
- **Composition:** CENTER. Provides `@ashlr/hub` + `ashlr` CLI, native MCP registry + gateway, Approval Inbox / proposal contract, `~/.ashlr/` shared state. Integrates external backends as one-config engines (Ollama/LM Studio, Claude Code CLI, Codex CLI, OpenAI/Anthropic-compatible APIs) + GitHub/Vercel/Telegram/iMessage/Raycast/Tauri.
- **Conventions:** ESM with explicit `.js` import specifiers (NodeNext). Layered: `cli/` thin handlers → `core/<domain>/` logic → `api/` curated re-exports (mutating/merge/apply paths intentionally unexported). All shared-state writes go to `~/.ashlr/`; CLI is sole writer. Tests flat in `test/` with milestone-prefixed names (h1.*, m45.*, m54.*), run serially for release. - **Status:** active / best-in-class-track — mature multi-crate workspace, 224 tests, full CI (build/test/clippy -D warnings/fmt), published to crates.io + npm + MCP/Smithery, E2E-encrypted cloud sync + team vaults shipping.
- **Key modules:** `crates/phantom-core/src` (config .phantom.toml, .env parse/rewrite, phm_ token gen 256-bit CSPRNG, HMAC-chain audit, rotation, importers, teams crypto, cloud sync); `phantom-vault/src` (VaultBackend trait: OS keychain + encrypted file argon2id fallback); `phantom-proxy/src` (hyper reverse proxy on 127.0.0.1: interceptor, body_scope, response_scrubber, rate_limiter); `phantom-cli/src/main.rs + commands/` (33 commands); `phantom-mcp/src/server.rs` (5190-line rmcp stdio server, 25 phantom_* tools); apps/web; supabase/migrations; npm wrappers.
- **Public API:** CLI `phantom` (init, exec, start/stop, list, add, remove, reveal, rotate, status, sync, team, export, import, audit, wrap/unwrap, …); MCP `phantom-mcp` (25 phantom_* tools — list_secrets, status, init, add_secret_interactive, rotate, cloud_push/pull, team_*; mutating tools require confirm:true, never accept plaintext values); Rust crates on crates.io (phantom-secrets-core/-vault/-proxy/-mcp); cloud HTTP API at phm.dev (/api/v1); npm packages.
- **Entry / build / test:** build `cargo build`; test `cargo test` (ignored e2e `-- --ignored`); lint `cargo clippy --all-targets -- -D warnings`; format `cargo fmt --all -- --check`; run `cargo run -- <command>`; install `npx phantom-secrets init`; CLI entry `phantom-cli/src/main.rs`; MCP entry `phantom-mcp/src/main.rs`; web `apps/web` next.
- **Composition:** Provides the fleet real API-key injection for engines/backends — fleet config + logs carry only `phm_` tokens (ecosystem bet #1). Provides 25 MCP secret tools to any MCP client. Consumed by ashlr-hub (e.g. NVIDIA NIM key via `phantom add`) and by **ashlr-stack as a hard dependency**. Consumes OS keychains, Supabase, Stripe, GitHub OAuth, Vercel. Distributed standalone via crates.io / npm / MCP registry / Smithery / Homebrew.

### binshield — *stops malicious deps before they ship*

## Repo organization — top improvements (prioritized)

Aggregated from the per-repo orgQuality findings. Themes recur across the platform: **doc-vs-code drift** (stale counts), **flat god-directories/files** that raise agent navigation + context cost, and **stray root cruft**.

1. **Kill doc-vs-code count drift platform-wide (highest leverage, affects 6+ repos).** README/CLAUDE.md headline numbers are stale and actively mislead agents: core-efficiency README claims ~17 tests / smaller genome (actual 55 tests, 24-file genome with quantized-ANN/rerankers); ashlrcode disagrees with itself (367 vs 689 tests, 42 vs 45 tools, v2.0/2.1); binshield tables list 5 packages (actual 10) and "31 endpoints" (actual ~82); ashlr-stack says 29/23 providers (actual 43) and 17 MCP tools (actual 19); morphkit says "9 MCP tools" (actual 10, morphkit_feature_context undocumented); ashlr-pulse README says v0.2 while shipping the full fleet plane (agent v0.4.0). Fix: generate counts/lists from code (a SOURCE-OF-TRUTH manifest consumed by README/site/docs) so they can't drift; add a CI check.

2. **Decompose monolithic files + flat god-directories for agent navigability + lower per-edit context cost (affects most repos).** ashlr-hub: `cli/index.ts` 68KB switch, `core/types.ts` 155KB, `run/orchestrator.ts` 96KB, `inbox/merge.ts` 69KB, `daemon/loop.ts` 66KB. ashlr-plugin: `servers/` flat ~120 files (group into tools/genome/stats/llm/). ashlrcode: `src/agent/` 64 files mixing stable + experimental. ashlr-pulse: `lib/` ~85 flat files (group fleet-/peer-share-/anomaly-). binshield: `apps/worker/src/` ~70 flat files (extend the ai-classifiers//analyzers/ pattern to feeds/pypi/alerts/). ashlr-stack: `core/src/` ~30 flat modules (a provisioning/ subdomain). webfetch: `cli/commands.ts` ~2500 lines. morphkit: `swiftui/layouts.ts` 3510 + `builder.ts` 2672. ashlr-md: `src/lib/` flat ~80 files. Fix: subgroup directories by domain + split the largest files.
